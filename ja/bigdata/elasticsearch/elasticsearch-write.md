# Elasticsearchの書き込み処理

単一のドキュメントの作成: インデックス リクエスト

複数のドキュメントの作成: 一括リクエスト

単一のドキュメントを作成するための操作プロセス:

![](/bigdata/elasticsearch/img.png)

1. クライアントは書き込みリクエストをノード 1 に送信します。

2. ドキュメント ID に基づいて、それが属するシャード 0 を特定します。コンテンツルーティングテーブル情報を通じてノード 3 でメインシャードを見つけた後、リクエストはノード 3 に転送されます。

3. ノード 3 で書き込み操作を実行します。プライマリ シャードの書き込みが成功すると、セカンダリ シャードに通知されます。すべてのセカンダリ シャードは正常に書き込まれ、調整ノードに成功を報告します。調整ノードは、書き込みの成功をクライアントに報告します。

書き込み一貫性ポリシーのデフォルトはクォーラムです。
quorum  = int((primary+number_of_replicas)/2) +1
